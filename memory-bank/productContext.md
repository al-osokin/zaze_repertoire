# Контекст продукта "Администрирование Афиши Театра"

## Почему этот проект существует:
Театр "Зазеркалье" нуждается в более эффективном способе управления составом исполнителей для своих спектаклей и автоматизации процесса создания карточек для социальных сетей (в частности, VK). Ручное заполнение состава является трудоемким и подверженным ошибкам процессом.

## Проблемы, которые он решает:
*   **Ручное заполнение состава:** Устранение необходимости вручную вводить артистов для каждой роли в каждом представлении.
*   **Несогласованность данных:** Обеспечение единообразия в именах артистов и ролей.
*   **Трудоемкость генерации карточек:** Автоматизация создания вики-разметки для карточек спектаклей.
*   **Отсутствие истории назначений:** Ведение истории назначений артистов на роли для предзаполнения и анализа.

## Как он должен работать:
Система должна предоставлять администратору удобный интерфейс для:
1.  Загрузки расписания с сайта театра.
2.  Сопоставления событий с существующими спектаклями.
3.  Просмотра и редактирования состава для каждого конкретного представления.
4.  Автоматического формирования вики-разметки карточки спектакля с актуальным составом.
5.  Учета различных типов исполнителей (актеры, дирижеры, пианисты).
6.  Запоминания "частых" исполнителей для каждой роли.

## Цели пользовательского опыта:
*   **Интуитивно понятный интерфейс:** Администратор должен легко ориентироваться в системе и выполнять свои задачи.
*   **Минимизация ручного ввода:** Максимальная автоматизация процессов.
*   **Быстрая генерация карточек:** Возможность быстро получать готовые карточки для публикации.
*   **Надежность данных:** Уверенность в корректности и актуальности информации о составе.
