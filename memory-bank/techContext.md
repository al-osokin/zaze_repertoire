# Технический контекст проекта "Администрирование Афиши Театра"

## Используемые технологии:
*   **Язык программирования:** PHP 8.1+
*   **База данных:** MariaDB (MySQL-совместимая)
*   **Веб-сервер:** Предполагается Apache/Nginx с поддержкой PHP
*   **Фронтенд:** HTML, CSS (возможно, с Tailwind CSS), JavaScript
*   **Библиотеки/расширения PHP:** `pdo_mysql`, `curl`, `dom`

## Среда разработки:
*   **IDE:** Visual Studio Code с плагином Cline
*   **Файловая система:** Работа с файлами на Google Drive (через синхронизацию или прямое подключение)
*   **Виртуальная среда:** WSL (Windows Subsystem for Linux) для выполнения PHP-скриптов и команд `mariadb`.

## Технические ограничения и особенности:
*   **Открытие браузера:** Команда `open http://...` не работает в WSL. Для проверки веб-интерфейса необходимо запускать PHP-сервер и открывать URL вручную в браузере Windows.
*   **Выполнение SQL-запросов:** Прямая передача запросов через командную строку `mysql -e "..."` может приводить к синтаксическим ошибкам. Рекомендуется использовать SQL-файлы для выполнения запросов `mariadb -u your_user -p avo_zaze < your_sql_script.sql`.
*   **Кодировка:** Использование `utf8mb4` для корректной работы с кириллицей и специальными символами.

## Зависимости:
*   **База данных:** Наличие работающего сервера MariaDB/MySQL.
*   **PHP:** Установленный PHP 8.1+ с необходимыми расширениями.
*   **Веб-сервер:** Настроенный веб-сервер для обслуживания PHP-приложения.

## Паттерны использования инструментов:
*   **`read_file` / `write_to_file` / `replace_in_file`:** Для работы с исходным кодом и файлами конфигурации.
*   **`execute_command`:** Для выполнения PHP-скриптов, команд `mariadb`, создания директорий и других системных операций.
*   **`list_files` / `search_files`:** Для навигации по проекту и поиска информации.
*   **`browser_action`:** Для взаимодействия с веб-интерфейсом (запуск, навигация, клики, ввод текста, снятие скриншотов).
